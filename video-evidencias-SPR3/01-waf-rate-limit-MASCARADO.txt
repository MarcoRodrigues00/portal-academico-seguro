== WAF — tentativa de SQL Injection ==

== WAF — tentativa de XSS ==
$ curl -k -s -o /dev/null -w "%{http_code}\n" "https://<HOST-DMZ-WAF>/?q=%3Cscript%3Ealert(1)%3C/script%3E"
403   <-- BLOQUEADO pelo WAF (OWASP CRS)

== Rate limiting — 12 requisições sequenciais a /api/login ==
$ for i in $(seq 1 12); do curl -k -s -o /dev/null -w "%{http_code} " "https://<HOST-DMZ-WAF>/api/login"; done
502 502 502 502 502 502 429 429 429 429 429 502
                        ^^^^^^^^^^^^^^^^^^^^^^^ 5 respostas 429 = rate limit (express-rate-limit) ativado
(502 nas demais: mesmo motivo do baseline — upstream fora do ar; rate limit já atua
na camada de proxy/WAF antes de tentar alcançar o upstream)

== Resultado resumido ==
- SQLi  -> 403 (bloqueado)
- XSS   -> 403 (bloqueado)
- Rate limit -> 429 obtido após repetição de requisições (limite ativo)
